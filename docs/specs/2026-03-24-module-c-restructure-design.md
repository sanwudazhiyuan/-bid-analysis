# 模块C重构：评分与商务综合模块设计规格

## 背景

当前模块C（评标办法与评分标准）使用平铺结构输出（C1评标方法 → C2评分细则 → C3价格公式），
无法准确反映招标文件中评分标准的层级结构。用户要求：

1. 严格按照原文子标题层级解析（支持 C2.1、C2.2 等二级编号）
2. 报价评分、商务评分、技术评分分开为独立大类
3. 将原"D.商务评分模块"中的报价要求、报价明细、后评价管理、分配规则并入模块C
4. 交叉引用（"详见评标办法前附表"、"据第X.X款规定"）需内联展开为表格

模块D（合同主要条款）保持现有固定模板结构不变。

## 目标输出结构

```
C. 技术评分模块
├── C1. 评分分值构成                    (standard_table)
├── C2. 报价评分标准（XX分）
│   ├── C2.1 子评分项1                  (standard_table)
│   ├── C2.2 子评分项2                  (standard_table)
│   └── ...
├── C3. 商务评分标准（XX分）
│   ├── C3.1 子评分项1                  (standard_table + 备注)
│   ├── C3.2 子评分项2                  (standard_table + 备注)
│   └── ...
├── C4. 技术评分标准（XX分）
│   ├── C4.1 子评分项1                  (standard_table + 备注)
│   └── ...
├── C5. 评分相等时的优先顺序            (standard_table)
├── C6. 报价要求                        (key_value_table)
├── C7. 报价明细（单价最高限价）        (standard_table)
├── C8. 后评价管理
│   ├── C8.1 评价指标                   (standard_table)
│   └── C8.2 评价运用                   (standard_table)
└── C9. 分配规则                        (standard_table)
```

注意：C1-C9 的具体标题和数量取决于实际招标文件内容，上述为典型结构。
LLM 应根据原文实际章节灵活输出，不硬编码固定数量。

### 字段说明

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | string | 编号，支持两级：C1, C2, C2.1, C2.2 等 |
| `title` | string | 严格跟随原文子标题，包含分值（如有） |
| `type` | string | `parent` / `standard_table` / `key_value_table` / `text` |
| `columns` | string[] | 表头列名 |
| `rows` | string[][] | 数据行 |
| `sections` | section[] | 仅 `type: "parent"` 时存在，包含子节 |
| `note` | string | 可选，证明材料要求或重要提示，渲染在表格下方 |

## 交叉引用处理

### 预处理阶段（Python `_filter_paragraphs` 增强）

在 `module_c.py` 的段落筛选中增加引用检测逻辑：

```python
# 引用检测正则
_REF_PATTERNS = [
    r"详见[《「]?(.+?)[》」]?",           # "详见评标办法前附表"
    r"按照?[《「]?(.+?)[》」]?的?规定",    # "按照XX的规定"
    r"据第\s*(\d+\.?\d*)\s*款",           # "据第2.1款"
    r"见附[件表录]\s*[《「]?(.+?)[》」]?", # "见附件XX"
    r"参照[《「]?(.+?)[》」]?",           # "参照XX"
]
```

处理流程：
1. 对已筛选段落的文本执行引用检测
2. 提取引用目标名称（章节标题/附表名/条款编号）
3. 在全文档段落中搜索匹配的目标段落
4. 将匹配到的段落追加到筛选结果中（去重）

### LLM 端（Prompt 指示）

在 prompt 中明确指示：
- 当原文中出现"详见XX"、"据第X款规定"等引用时，在输入段落中查找被引用的内容
- 将被引用内容直接展开为表格，嵌入到引用位置
- 如果引用的内容在输入中找不到，保留原文引用文字并标注 `[未找到引用内容]`

## 报告生成器适配

### `_render_sections` 多级编号支持

当前 `report_gen.py` 的 `_render_sections` 使用 `{letter}.{idx}` 格式编号。
需要修改为支持递归编号：

- 顶层：C1, C2, C3...
- 嵌套层：C2.1, C2.2, C2.3...

递归调用时传递父级编号前缀，子节编号为 `{parent_prefix}.{child_idx}`。

### `note` 字段渲染

当 section 包含 `note` 字段时，在表格下方添加一个灰色小字段落，格式：
```
证明材料要求：XXX
```
或
```
重要提示：XXX
```

## 涉及修改的文件

| 文件 | 改动类型 | 说明 |
|------|----------|------|
| `config/prompts/module_c.txt` | 重写 | 新prompt支持嵌套子标题、交叉引用展开、报价/商务/技术分离 |
| `src/extractor/module_c.py` | 增强 | `_filter_paragraphs` 增加引用检测和附表段落追加；扩大关键词范围覆盖报价要求、后评价等内容 |
| `src/generator/report_gen.py` | 增强 | `_render_sections` 支持多级编号（C2.1格式）；新增 `note` 字段渲染 |

| 文件 | 说明 |
|------|------|

| `src/extractor/base.py` | 基础设施无需改动 |

## 兼容性考虑

- JSON 输出格式向后兼容：新增 `parent` type 和 `note` 字段，已有的 `standard_table` / `key_value_table` / `text` 类型不变
- `report_gen.py` 的递归 `_render_sections` 已支持 `sections` 嵌套，只需调整编号逻辑
- 模块D完全不受影响
